#yii自动加载的机制有两种，一种是通过yii类里面的classmap属性注册，一种是yii类里面的alias属性注册 #第一种只能一次加载一个类，第二种alias的可以把整个文件夹下的类都注册了 classmap这边注册的核心命名空间有19种[没有yii\test] 其中带s的五种，yii\widgets , behaviors ,filters,helpers,validators 基类命名空间两个yii\base web 框架都具有的基础性能7个 db data log i18n mail caching capcha yii特性的5个 rest auth grid rbac di 另外还有11个alias注册的命名空间 其中和模块相关的有4个 common frontend backend app 和目录结构相关的5个 runtime web webroot vendor console 两个包命名空间 yii kartik 还有yii中核心的component组件有16个 其中5个和classmap同名的， db cache i18n log maile 5个比较常用的request response user session view 3个名字带manager的 urlManager authManager assetManager 3个不常用的security formatter errorHandle #yii autoload机制 Yii中使用BaseYii::autoload($classname)来按需加载类， 满足以下规则的一种： 1.查找调用的类是不是在Yii::$classMap这个数组中， 你可以参考Yii::$classMap['yii\base\Action'] = FILE_PATH.'/base/Action.php'来注册需要自动加载的类; FILE_PATH可以是绝对路径，也可以是alias形式(带@)的路径 2.如果不在classmap中但是有命名空间的话，Yii会 默认将namespace(即第一个/前面的部分)加@， 去查getAlias得到的类，如果文件存在即加载 Yii::setAlias('myNamespace',dirname('.').'/path1/myNamespace'); ...该语句将根命名空间和绝对路径关联... 总结，如果想使用yii的自动加载，可以采用注册classMap的方式将use的部分作为键， 文件绝对路径作为值(也可以是已经注册过的alias变量拼接的路径)，或者采用将根命名空间注册为alias的方式， classmap适合加载单个文件，alias适合整个文件夹的文件都加载 #Yii2自带的19个命名空间及作用[不包含test]，classmap方式加载的类 #用法use yii\base\View;查看classes.php可获得 base 核心基类命名空间 常用 Exception Theme Action Component Model Module Object Behavior
